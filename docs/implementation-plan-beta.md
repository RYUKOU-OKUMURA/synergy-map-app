# Beta 実装計画

作成日: 2026-05-16

## 目的

Betaでは、MVP-1で確認した最小価値を実務利用に耐える品質へ拡張する。MVP-1の結果によって優先順位は変わるため、この計画は詳細化しすぎず、主要テーマとタスク候補を整理する。

## Beta開始条件

- [ ] MVP-1の中心導線が通っている。
- [ ] 実資料でAI抽出カードと顧客導線マップの価値が確認できている。
- [ ] source chunks、item_sources、ai_runsが実務確認に使えている。
- [ ] Markdown / CSV出力が使われている。
- [ ] PDFを継続するか、Markdown中心に寄せるか判断できている。

## B1 事業・収益ビュー

- [ ] 事業・収益ビューの目的を再定義する。
- [ ] 顧客導線ビューと共通利用するnodes / edgesを整理する。
- [ ] ビュー固有のlayoutを保存できるようにする。
- [ ] 売上・購買の流れを中心にしたedge表現を作る。
- [ ] 財務参考情報ノードの表示を強化する。
- [ ] 顧客導線ビューとの切り替えUIを作る。
- [ ] 同じデータから2ビューを生成するAI依頼を作る。

完了条件:

- [ ] 顧客導線ビューと事業・収益ビューを切り替えて確認できる。

## B2 PDF出力強化

- [ ] MVP-1のPDF利用状況を確認する。
- [ ] レポート型PDFのテンプレートを改善する。
- [ ] ダッシュボード型PDFを追加する。
- [ ] 情報充実度の表現を整える。
- [ ] 優先施策一覧を見やすくする。
- [ ] マップ画像の解像度と余白を改善する。
- [ ] PDF出力時のフォント差を解消する。
- [ ] PDFプレビューを追加するか判断する。

完了条件:

- [ ] 会議・提案でそのまま使えるPDFを生成できる。

## B3 バージョン差分比較

- [ ] 名前付きバージョン保存を実装する。
- [ ] バージョン一覧を表示する。
- [ ] 2つのバージョンを選択できる。
- [ ] 差分分類を定義する。
  - [ ] 追加
  - [ ] 削除
  - [ ] 変更
  - [ ] 強化
  - [ ] 弱体化
- [ ] `change_events`または`version_items`を追加するか判断する。
- [ ] マップ差分表示を作る。
- [ ] AIによる比較コメントを生成する。
- [ ] 比較レポートMarkdown / PDFを出力する。

完了条件:

- [ ] 初回分析版と現在版の変化を説明できる。

## B4 本格Codex相談UI

- [ ] 案件内チャットUIを作る。
- [ ] thread resumeのUXを整える。
- [ ] 現在のマップ、抽出カード、施策、source chunksを文脈として渡す。
- [ ] ユーザーの質問テンプレートを用意する。
- [ ] Codexの提案を「反映候補」として表示する。
- [ ] DB変更前にユーザー承認を必須にする。
- [ ] turn interruptionをUIから実行できるようにする。

完了条件:

- [ ] コンサルタントが案件について追加相談し、変更提案を承認して反映できる。

## B5 会議中モード

- [ ] 会議中モードの画面を定義する。
- [ ] 長い分析より短い再分析を優先する。
- [ ] ノード名、分類、線の状態、優先度を素早く編集できるようにする。
- [ ] 確認質問を追加・完了できるようにする。
- [ ] 会議メモを残せるようにする。
- [ ] 会議後にスナップショットを保存する。

完了条件:

- [ ] 会議中に操作しても迷わず、短時間で変更と再分析ができる。

## B6 OCR / URL読み取り

- [ ] スキャンPDFの発生頻度を確認する。
- [ ] OCRをローカルで行うか、外部APIで行うか判断する。
- [ ] HP URL読み取りの範囲を決める。
- [ ] SNS URL読み取りの範囲を決める。
- [ ] robots / 利用規約 / 認証が必要な情報を扱わない方針を決める。
- [ ] OCR / URL由来の情報を公開情報として区別する。

完了条件:

- [ ] OCRまたはURL読み取りを入れるか、明確に後回しにするか判断できている。

## B7 配布・更新

- [ ] Tauri updaterを採用するか判断する。
- [ ] Windows署名の方針を決める。
- [ ] macOS notarizationの方針を決める。
- [ ] Codex App Server sidecarの更新方式を決める。
- [ ] Typst sidecarの更新方式を決める。
- [ ] ログ収集を入れる場合、機密本文を除外する。

完了条件:

- [ ] Windows / macOS利用者に配布できる現実的な手順がある。

## B8 クラウド保存・同期

- [ ] ローカルファーストを維持した同期方針を検討する。
- [ ] Google Drive / OneDrive / Dropbox保存の優先度を判断する。
- [ ] 同期対象を分ける。
  - [ ] SQLite snapshot
  - [ ] 元資料
  - [ ] exports
  - [ ] Codex thread metadata
- [ ] クラウド保存時の情報管理方針を決める。

完了条件:

- [ ] クラウド保存を入れるか、エクスポート運用で十分か判断できている。

## B9 実務品質改善

- [ ] 実ユーザーの操作ログではなく、手動ヒアリングで詰まりを集める。
- [ ] 抽出カードの編集体験を改善する。
- [ ] マップの見やすさを改善する。
- [ ] 施策カードの優先度表現を改善する。
- [ ] 情報充実度の算出・表示を改善する。
- [ ] 大きい案件でのパフォーマンスを改善する。

完了条件:

- [ ] 継続利用時の不満点が整理され、優先度順に改善されている。

## Beta完了判定

- [ ] 事業・収益ビューまたはPDF強化のどちらかが実務で使える水準になっている。
- [ ] バージョン差分比較の初期版が動く、または明確に後回し判断されている。
- [ ] 本格Codex相談UIの必要性が検証されている。
- [ ] 配布・更新の現実的な方針がある。
- [ ] 次の正式版ロードマップを作れる状態になっている。
