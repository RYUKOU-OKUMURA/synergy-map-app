# MVP-1 Spec

## Core Flow

1. 資料を入れる。
2. アプリが資料をsource chunksと出典情報に分解する。
3. Codexが事業・商品・チャネル・顧客接点・財務参考情報を抽出する。
4. 抽出結果をカードで確認・修正・確定する。
5. 顧客導線ビューのシナジーマップを生成する。
6. AIが短い現状分析、簡易施策、確認質問を出す。
7. Markdown、CSVで出力する。

## MVP-1 Must-Haves

- コンサル側専用の実務ツールとして作る
- クライアント共有はMarkdown/CSV出力を必須にし、PDFはレポート型1テンプレートのみ検証後に対応
- 手動アップロード資料を優先
- Codex App ServerによるAI抽出・マップ生成・簡易施策生成を検証
- AI実行履歴を保存する
- source chunks / item_sourcesで出典を追えるようにする
- 情報充実度、確定/推定/要確認を明示
- ノードの影響力は、売上・利益貢献、顧客接点の強さ、現場の主観重要度を重視
- 線は、顧客の流れ、売上・購買の流れ、認知・信頼の流れを表す
- 会議中は軽い修正と短いAI再分析を優先

## MVP-1 Out of Scope

- クライアントログイン
- 共同編集
- Web共有リンク
- Google Analytics / YouTube Studio / SNSインサイトの認証連携
- freee / マネーフォワード等のAPI連携
- Google Drive / OneDrive / Dropbox保存
- Word出力
- 3Dマップ
- 自由配置型の高度な作図編集
- 事業・収益ビュー
- ダッシュボード型PDF
- 複数PDFテンプレート
- バージョン差分比較
- 本格的なCodex相談チャットUI
- OCR
- URL読み取り
