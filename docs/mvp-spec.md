# MVP-1 Spec

## Core Flow

1. マップの材料を入れる。メモだけでも始められ、資料がある場合は追加できる。
2. アプリが資料やメモをsource chunksと出典情報に分解する。
3. Codexが事業・商品・チャネル・顧客接点・財務参考情報を抽出する。
4. 抽出結果をカードで確認・修正・確定する。
5. 顧客導線ビューの売上マップを生成する。
6. AIが考える材料、売上の流れ、詰まり、確認質問、次に試す一手を整理する。
7. 自分が次に考える/動くための記録としてMarkdown、CSVで出力する。

## MVP-1 Goal

MVP-1は、見やすい売上マップが出ることを最低ラインにする。そのうえで、主役ユーザーが自分の事業の詰まり、未接続の可能性、次に試す一手、追加で確認すべきことを判断できる状態をゴールにする。

## MVP-1 Must-Haves

- 主役ユーザーは個人事業主・小規模会社の経営者とし、代表ユーザーは奥村さん自身とする
- コンサルタント、AI顧問、経営支援者は副次ユーザーとして扱う
- メモだけでも仮説マップを作れる
- 手動アップロード資料も追加材料として扱える
- Codex App ServerによるAI抽出・マップ生成・簡易施策生成を検証
- AI実行履歴を保存する
- source chunks / item_sourcesで出典を追えるようにする
- 情報充実度、確定/推定/要確認を明示
- ノードの影響力は、売上・利益貢献、顧客接点の強さ、現場の主観重要度を重視
- 線は、顧客の流れ、売上・購買の流れ、認知・信頼の流れを表す
- AIは答えを完成させるのではなく、思考・判断できる準備状態を作る
- Markdown/CSV出力は、自分が次に考える/動くための記録を優先する

## MVP-1 Out of Scope

- クライアントログイン
- 共同編集
- Web共有リンク
- Google Analytics / YouTube Studio / SNSインサイトの認証連携
- freee / マネーフォワード等のAPI連携
- Google Drive / OneDrive / Dropbox保存
- Word出力
- 3Dマップ
- 自由配置型の高度な作図編集
- レイヤー型の深掘りマップ
- 本格的な施策評価ビュー
- ダッシュボード型PDF
- 複数PDFテンプレート
- レポート型PDF出力
- バージョン差分比較
- 本格的なCodex相談チャットUI
- 常時AIが横から提案するモード
- OCR
- URL読み取り
