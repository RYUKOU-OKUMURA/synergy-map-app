# Phase 1 試験運用チェックリスト

作成日: 2026-05-31

## 目的

開発者本人がmacOS上の`pnpm tauri dev`で、自分の事業整理・思考整理に継続利用できる最低ラインを確認する。

## 完成条件

- [ ] `pnpm format:check`が通る。
- [ ] `cd src-tauri && cargo fmt -- --check`が通る。
- [ ] `pnpm lint`が通る。
- [ ] `pnpm build`が通る。
- [ ] `cd src-tauri && cargo test`が通る。
- [ ] `pnpm tauri dev`で起動し、既存データが再起動後も復元される。

## 日常利用フロー

- [ ] 新規マップを短いメモだけで作成できる。
- [ ] AI送信前確認で、AIへ渡る要約と送信範囲を確認できる。
- [ ] AI抽出を実行し、抽出カードを確認・編集できる。
- [ ] 売上マップを生成し、ノード/導線の意味を確認できる。
- [ ] 次の一手と確認質問を生成できる。
- [ ] AI生成の確認質問が「記録」ビューの確認事項にも登録される。
- [ ] 手動で確認事項を追加、編集、完了、見送りにできる。
- [ ] 思考メモ、会議メモ、日次メモを追加・編集・削除できる。
- [ ] History画面で現在の状態を名前付き保存し、一覧で確認できる。

## 情報ソース

- [ ] PDFを追加し、chunks/source previewを確認できる。
- [ ] CSVを追加し、chunks/source previewを確認できる。
- [ ] XLSXを追加し、chunks/source previewを確認できる。
- [ ] Markdownを追加し、chunks/source previewを確認できる。
- [ ] Textを追加し、chunks/source previewを確認できる。
- [ ] 情報ソース単体を削除できる。
- [ ] 削除後、既存カード/マップは残り、再抽出/再生成推奨の通知が出る。

## AI接続

- [ ] Codex接続確認が通る。
- [ ] Composer接続確認が通る。
- [ ] AIプロバイダをCodexへ切り替えて抽出/生成を確認する。
- [ ] AIプロバイダをComposerへ切り替えて抽出/生成を確認する。
- [ ] 失敗時フォールバックのON/OFFを保存できる。

## 出力

- [ ] 設定画面で既定出力フォルダを選択できる。
- [ ] Markdownを設定済みフォルダへ出力できる。
- [ ] CSV bundleを設定済みフォルダへ出力できる。
- [ ] 設定済みフォルダが使えない場合、アプリ内exportsへフォールバックし警告が出る。
- [ ] 出力履歴から出力先を開ける。

## 実事業メモでの価値確認

- [ ] 直近の事業メモを1件投入し、売上導線が自分の認識と照合できる。
- [ ] 「今確認すべきこと」が具体的に1つ以上残る。
- [ ] 「次に試す一手」が実行可能な粒度で1つ以上残る。
- [ ] Markdown/CSV出力が月次振り返りや相談準備に使える形になっている。
