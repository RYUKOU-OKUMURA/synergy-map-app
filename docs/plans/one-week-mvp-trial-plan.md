# 1週間 MVP 実運用チェックリスト

作成日: 2026-06-02

## 目的

Windows通し確認は未実施のまま、開発者本人がmacOSで1週間の実事業運用に入れるかを確認する。

このチェックリストは、新機能を増やすためではなく、MVP-1の中心価値が日常運用で壊れないかを確認するために使う。

中心価値:

> マップの材料入力 → AI抽出カード → 人間確認 → 顧客導線の売上マップ → 次に考える/動くためのMarkdown / CSV出力

## 実運用開始前の判定

### すでに満たしていること

- [x] `pnpm format:check`が通る。
- [x] `cd src-tauri && cargo fmt -- --check`が通る。
- [x] `pnpm lint`が通る。
- [x] `pnpm build`が通る。
- [x] `cd src-tauri && cargo test`が通る。
- [x] マップの材料入力からMarkdown / CSV出力までの主要実装がある。
- [x] AI抽出カードを人間が確認、編集、確定できる。
- [x] 顧客導線の売上マップを生成、軽微編集できる。
- [x] 出典、AI実行履歴、保存済みスナップショットを追える。
- [x] APIキーや認証情報をSynergy Map本体に保存しない方針が実装、記録されている。

### 実運用前に確認すること

- [ ] `pnpm tauri dev`で起動できる。
- [ ] 既存プロジェクトを開くと「今日」ビューから再開できる。
- [ ] 設定画面でAIプロバイダ、フォールバック、出力先を確認できる。
- [ ] CodexまたはComposerのどちらかで接続確認が通る。
- [ ] 短い実事業メモを1件投入し、AI抽出、マップ生成、次の一手生成まで通る。
- [ ] Markdown / CSVを任意の出力先へ保存し、出力履歴から開ける。

## 1週間で毎日見ること

- [ ] その日に追加したいメモ、違和感、商品情報、集客情報を「情報ソース」または「記録」に入れる。
- [ ] 「今日」ビューの次アクションが自然か確認する。
- [ ] 未完了の確認事項を1つ以上、完了または見送りにする。
- [ ] 必要に応じて思考メモを残す。
- [ ] 売上マップを見て、詰まり、未接続、次に試す一手が判断しやすいかをメモする。

## 1週間の終了判定

### MVPとして合格

- [ ] 1週間で3回以上、実事業メモまたは記録を追加した。
- [ ] 少なくとも1回、AI抽出から売上マップ生成まで通した。
- [ ] 少なくとも1回、次の一手を確認事項へ変換した。
- [ ] MarkdownまたはCSVを1回以上出力し、振り返りや相談準備に使えた。
- [ ] 売上マップを見て、詰まり、未接続、次に試す一手、確認すべきことを1つ以上判断できた。
- [ ] 再起動後も、今日ビュー、記録、履歴、出力履歴から前回状態を再開できた。

### 要改善

- [ ] 何を入力すればよいか迷う。
- [ ] AI送信前確認で、何が送られるか分かりにくい。
- [ ] 抽出カードの修正が面倒で使わなくなる。
- [ ] 売上マップを見ても、事業の流れや詰まりが判断できない。
- [ ] 次の一手が抽象的で、実際の行動に落ちない。
- [ ] 今日ビューを開いても、次に何をすればよいか分からない。
- [ ] Markdown / CSV出力が振り返りや相談準備に使いにくい。

## この1週間ではやらないこと

- Windows通し確認
- 配布、署名、notarization
- PDF強化
- OCR
- URL本文自動取得
- 本格チャットUI
- バージョン差分比較
- レイヤー型深掘りマップ
- Composer常駐セッション
- React component testの本格整備
- repository層の全面分離

## 既知のリスク

- Windowsでの通し確認は未実施。
- テキスト抽出できないPDFはOCR対象外のため、情報ソースとしては使えない。
- Codex threadのmap単位resumeは未実装。ただし、MVP-1の1週間個人運用では`ai_runs`、AIコメント、記録、履歴で追跡する。
- UIの細かい摩擦は、1週間の運用メモをもとに直す。
