# Phase 2 実事業試験運用チェックリスト

作成日: 2026-05-31

## 目的

開発者本人が実事業データを使い、週次/月次の事業整理で継続利用できる状態を確認する。

Phase 2では新しいDBテーブルを増やさず、既存の`action_items`、`map_notes`、`suggestions`、`versions`、`export_jobs`を使って、「今日見るべきこと」と「次に動くこと」に自然に着地する導線を整える。

## 完成条件

- [ ] `pnpm format:check`が通る。
- [ ] `cd src-tauri && cargo fmt -- --check`が通る。
- [ ] `pnpm lint`が通る。
- [ ] `pnpm build`が通る。
- [ ] `cd src-tauri && cargo test`が通る。
- [ ] `pnpm tauri dev`で起動し、既存プロジェクトを開いたときに「今日」ビューが初期表示される。
- [ ] 実事業メモで、入力、AI抽出、マップ生成、次の一手、確認事項、メモ、名前付き保存、Markdown/CSV出力まで通し運用できる。

## 今日ビュー

- [ ] プロジェクト内ナビに「今日」が表示される。
- [ ] 既存プロジェクト選択時の初期表示が「今日」になる。
- [ ] 新規マップ作成時は従来通りマップ作成フローが優先される。
- [ ] 情報がない状態では、情報追加が次アクションとして表示される。
- [ ] source chunksがあり抽出カードがない状態では、AI抽出が次アクションとして表示される。
- [ ] 抽出カードがありマップがない状態では、マップ生成が次アクションとして表示される。
- [ ] 追加・削除ソースや未反映カードがある状態では、再抽出/再生成推奨が表示される。
- [ ] 未完了確認事項がある状態では、確認事項の整理が次アクションとして表示される。
- [ ] 落ち着いた状態では、名前付き保存とMarkdown/CSV出力が次アクションとして表示される。

## 確認事項

- [ ] `action_items.status = open`の項目が優先度順に表示される。
- [ ] 「今日」ビューから確認事項を完了にできる。
- [ ] 「今日」ビューから確認事項を見送りにできる。
- [ ] 完了/見送り後、workspaceと表示が即時に更新される。
- [ ] 「記録」ビューでは既存通り確認事項を詳細編集できる。

## 次の一手

- [ ] 未却下の`suggestions`が「今日」ビューに表示される。
- [ ] 施策を確認事項へ変換できる。
- [ ] 変換した確認事項は`source_type = "suggestion"`、`source_id = suggestionId`で保存される。
- [ ] 同じsuggestion由来の未完了確認事項がある場合、重複作成されない。
- [ ] 存在しないsuggestionからの作成はエラーになる。

## 最近の状態

- [ ] 最近の思考メモ、会議メモ、日次メモを確認できる。
- [ ] 最近の名前付き保存を確認できる。
- [ ] 最近のMarkdown/CSV出力履歴を確認できる。
- [ ] 必要に応じて「記録」「履歴」「出力」ビューへ移動できる。

## 実事業シナリオ

- [ ] 直近の事業メモを投入し、抽出カードの要確認項目を整理できる。
- [ ] 売上マップを見て、詰まりまたは未接続の可能性を1つ以上説明できる。
- [ ] 次に試す一手を確認事項へ変換し、今日の確認対象として扱える。
- [ ] 思考メモを追加し、再起動後も「今日」ビューから最近の状態を確認できる。
- [ ] Markdown/CSV出力が月次振り返りや相談準備に使える内容になっている。
