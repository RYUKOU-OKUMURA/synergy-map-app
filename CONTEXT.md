# Synergy Map App Context

このコンテキストは、個人事業主・小規模会社の経営者が自分の事業構造を売上マップに整理するためのプロダクト用語を定義する。プロダクト、UX、実装判断で同じ言葉を使うために存在する。

## Language

**主役ユーザー**: 自分の事業を自分で整理したい個人事業主・小規模会社の経営者。MVPでは、この人の理解、入力しやすさ、判断の進みやすさを最優先する。
_Avoid_: コンサル専用ユーザー, クライアント分析者

**代表ユーザー**: 主役ユーザーを具体化する最初の基準人物としての奥村さん自身。MVP判断では、代表ユーザーが継続して使いたいかを重要な判定材料にする。
_Avoid_: ペルソナだけの架空ユーザー

**副次ユーザー**: 中小企業向けコンサルタント、AI顧問、経営支援者、士業、支援機関など、他者の事業整理を支援する人。MVPでは主役ユーザーの体験を崩さない範囲で考慮する。
_Avoid_: メインターゲット

**売上マップ**: 商品、集客、顧客接点、売上の流れを1枚で把握するための事業マップ。外向きの説明では「シナジーマップ」より優先する。
_Avoid_: 図解, マインドマップ, 資料マップ

**シナジーマップ**: プロダクト内部で使う、売上マップの専門用語。初見ユーザー向けコピーでは補助的に扱う。
_Avoid_: 初見ユーザーへの単独訴求

**マップの材料**: 売上マップを作るためにユーザーが入れる情報の総称。資料ファイルだけでなく、メモ、違和感、商品情報、集客状況、URL、SNS、売上の感覚値も含む。
_Avoid_: 資料だけ, アップロードファイルだけ

**仮説マップ**: 情報が少ない状態でも作る売上マップ。正確な分析結果ではなく、推定や要確認を含む考えるための初期地図として扱う。
_Avoid_: 確定分析, 正解マップ

**次に試す一手**: 売上マップを見たあとにユーザーが実行候補として持ち帰れる具体的な施策。売上改善を保証する答えではなく、判断を進めるための候補として扱う。
_Avoid_: 最終提案, 売上保証施策

**確認すべきこと**: マップの精度を上げるため、または施策判断の前にユーザーが追加で確かめる情報。AIの推測と事実を分けるための問いとして扱う。
_Avoid_: 宿題リストだけ, コンサル質問票だけ

## Flagged Ambiguities

**主ユーザー**: 以前の資料ではコンサルタント/支援者を主ユーザーとする記述がある。MVPの現在の解釈では、主役ユーザーは個人事業主・小規模会社の経営者であり、コンサルタント/支援者は副次ユーザーとする。

**資料**: 「資料」と言うとPDFやExcelだけに聞こえる。MVPの入口では、より広い「マップの材料」を使う。

## Example Dialogue

Developer: MVPでは、資料アップロードを最初の画面にしますか？

Domain Expert: 主役ユーザーは整理された資料を持っていないことが多いので、「マップの材料」を入れる入口にしてください。メモだけでも仮説マップを作れる必要があります。

Developer: 仮説マップは分析結果として扱いますか？

Domain Expert: いいえ。仮説マップは考えるための初期地図です。推定や要確認を明示し、次に試す一手と確認すべきことにつなげます。
