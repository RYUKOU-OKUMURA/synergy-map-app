# AGENTS.md

このリポジトリでCodexまたはサブエージェントが作業するときは、まず `docs/project-context.md` を読んでから、タスクに必要な関連資料だけを読む。

詳細な運用ルールは `agent.md` を参照する。

要点:

- すべての `docs/*.md` を毎回読まない。
- プロダクト判断やUX判断では `docs/product-vision-business-principles.md` と `docs/product-competitive-positioning-and-strategy.md` を参照する。
- MVP-1の実装判断では `docs/mvp-spec.md` と `docs/requirements.md` を参照する。
- 実装進捗は該当する `docs/implementation-plan-*.md` のチェックボックスを正本にする。
- ユーザーの既存変更を勝手に戻さない。
- MVP-1の範囲外機能を、明示的な指示なく実装しない。
